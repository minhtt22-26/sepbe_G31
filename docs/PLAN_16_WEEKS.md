# 16-week plan (team 5)

Assumption: 8 sprints × 2 weeks.

## Roles (recommended)

- TL/Architecture (1)
- Backend feature owners (3)
- QA/Automation + Release manager (1)

## Sprint outline

### Sprint 1 (Weeks 1–2): Foundation

- Repo conventions, env, Docker DB, Prisma baseline, auth skeleton, health.

### Sprint 2 (Weeks 3–4): Core domain v1

- User CRUD, validation, pagination patterns, error handling.

### Sprint 3 (Weeks 5–6): Auth v1

- JWT login/register, password hashing, guards, request user context.

### Sprint 4 (Weeks 7–8): Domain v2

- Add next core modules (based on your topic), relations in Prisma, services.

### Sprint 5 (Weeks 9–10): Integrations

- File storage/email/third-party APIs if required; background jobs if needed.

### Sprint 6 (Weeks 11–12): Hardening

- Observability (logs), rate limiting if needed, security review, test coverage.

### Sprint 7 (Weeks 13–14): UAT

- Fix bugs, refine API contracts, seed data, performance checks.

### Sprint 8 (Weeks 15–16): Release

- Deployment guide, final polish, demo scripts, final report.
